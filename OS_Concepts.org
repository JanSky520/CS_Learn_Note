* 导论
** 操作系统的功能
1. 操作系统分为四个组件：硬件、操作系统、用户程序、用户
2. 操作系统可以分为硬件、软件以及数据
3. 操作系统主要关注的是系统的易用性，次要关注性能和安全性，并不需要关注资源利用率
4. 可以把操作系统看作一个资源分配器，另一个视角可以把操作系统当成是个控制程序
5. 操作系统是一直运行在计算机上的程序（内核），其他两类程序是系统程序和应用程序
6. 移动操作系统不只有内核还有中间件，为应用程序开发人提供其他功能的软件框架
** 计算机系统的组成
1. 计算机系统包括一个或多个 CPU 和若干个设备控制器，通过共用总线相连，总线提供共享内存的访问
2. 设备控制器在所控制的外围设备与本地缓存之间进行数据传递，操作系统为设备控制器提供设备驱动程序，提供统一设备访问接口
3. 硬件可以通过系统总线随时发生信号到 CPU 触发中断，它是操作系统和硬件交互的关键部分
4. CPU 被中断时停止正在做的事，立即跳转到中断服务程序的开始地址执行，执行完成后继续中断前的操作
5. 通过指针表可以间接调用中断程序，指针表位于低地址内存中，保存各种设备的中断处理程序地址
6. CPU 执行每条命令后都会检测中断请求线，检测到中断信号后会读取中断号码作为中断向量的索引来跳转到中断处理程序
7. 非屏蔽中断通常是发生紧急情况需要紧急处理的中断，可屏蔽中断是可临时被屏蔽的不重要的中断
8. 中断链技术允许多个中断处理程序共享一个中断号，触发中断后会依次查询是哪条具体的中断
9. 中断优先级系统使 CPU 延迟处理低优先级中断而不屏蔽所有中断，并且让高优先级中断抢占低优先级中断
10. 程序必须在内存中才能被执行，CPU 从内存中加载指令并存到指令寄存器，接着指令被解码执行
11. RAM 断电后会丢失数据，数据主要被保存在外存中
** 计算机系统的体系结构
1. 处理核是执行指令和利用寄存器存储数据的组件，带处理核的 CPU 可以执行通用指令包括进程的指令
2. 多处理器共享计算机总线、时钟、内存和外围设备，可以同时运行多个程序
3. 非均匀内存访问是一种多处理器架构，其中CPU访问本地内存速度快于远程内存，需通过软件优化减少跨节点访问以提升性能
4. 松耦合指系统或组件之间的依赖关系较弱，彼此通过标准化接口交互，而非紧密绑定，通过降低依赖提升系统的可维护性和可扩展性
5. 集群计算机共享存储，采用局域网或使用更快的内部连接
6. 集群通常用于高可用性服务，集群中的一个或多个系统出错仍可继续提供服务
7. 并行计算指将一个程序分成多个部分，每个部分可以并行运行在计算机或集群计算机的各个核上
** 操作系统的执行
1. 计算机启动时需要有一个初始程序才能运行，通常以固件的形式存储在计算机硬件中
2. 有些服务由系统程序在内核外提供，在启动时加载到内存中成为系统守护进程，linux 上的第一系统程序是 systemd
3. 陷阱是程序的主动求助，而异常是被动事故；两者均通过中断机制进入内核，但陷阱是预期行为，异常可能致命
4. 多道程序设计通过组织程序使得 CPU 总有一个在执行作业，提高 CPU 利用率，正在执行的程序成为进程
5. 多任务是多道程序设计的逻辑扩展，CPU 在多个进程之间切换来执行多个进程
6. 通常计算机分为用户模式和内核模式，计算机硬件通过一个模式位来表示当前模式
7. 系统引导时硬件从内核模式开始，引导完成后切换到用户模式；有陷进或中断会进入内核模式，执行后回到用户模式
8. 操作系统应维持控制 CPU，防止用户程序进入死循环，使用定时器触发中断自动跳转到内核模式
