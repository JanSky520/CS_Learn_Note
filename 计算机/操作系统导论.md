# 虚拟化
+ 进程：程序的一次执行过程，是系统进行资源分配和调度的基本单位 
+ 线程：一个进程至少包含一个线程（主线程），也可以包含多个线程，线程是CPU调度的基本单位，这意味着CPU真正执行的是线程不是进程
+ 协程：协程的创建、切换、销毁完全在用户空间完成，开销低
+ 时分共享：一个进程只运行一个时间片，然后切换到其他进程，操作系统提供了存在多个虚拟 CPU 的假象
+ 空分共享：将资源从空间上划分为多个独立的、互不重叠的部分，并同时分配给不同的使用者，独占属于自己的的空间
+ 进程 API：创建、销毁、等待、其他控制、状态
+ 进程创建：分配唯一 PID 并创建 PCB，将代码和静态数据加载到内存，并为堆栈分配初始空间，把新进程的 PCB 放入就绪队列等待 CPU 调度
+ 进程状态：创建、就绪、运行、阻塞、终止
+ 僵尸进程：已经终止运行，但仍然保留 PCB，等待父进程回收
+ PCB：是内核中一个结构体，它记录了进程的 ID、状态、寄存器、内存地址、打开文件等所有信息，是操作系统管理进程的唯一依据
+ 机制：系统为实现某种功能而提供的底层操作、数据结构或硬件支持（如何做）
+ 策略：系统在特定目标下，决定如何使用机制的规则、算法或决策逻辑（做什么）
+ 系统调用：为用户程序提供一组受控的、安全的接口，允许用户程序操作系统内核运行更高权限的资源，是用户态主动进入内核态的唯一标准入口
+ 陷阱：处理器的一种同步机制，由正在执行的程序主动触发，从用户态切换到内核态，处理完后返回用户态
+ 中断：处理器的一种异步响应机制，由外部硬件或内部异常条件触发，暂停当前执行序列，执行预设中断处理程序，完成后恢复
+ 上下文切换：操作系统内核将 CPU 从一个进程切换到另一个进程，保存当前进程的执行状态，恢复目标进程的先前状态
+ 进程调度策略：先到先服务（FCFS）、短作业优先（SJF）、优先级调度、时间片轮转（RR）、多级反馈队列（MLFQ）
