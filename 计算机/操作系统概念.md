# 导论
+ 总线：总线是连接计算机系统中多个部件（如 CPU、内存、I/O 设备）之间进行数据传输的一组公共通信线路
+ 中断：处理器的一种异步响应机制，由外部硬件或内部异常条件触发，暂停当前执行序列，执行预设中断处理程序，完成后恢复
+ 陷阱：处理器的一种同步机制，由正在执行的程序主动触发，与中断一样从用户态切换到内核态，处理完后返回用户态
+ 直接内存访问（DMA）：允许硬件设备绕过 CPU，直接与系统内存进行数据传输
+ 系统守护进程：在后台运行、没有控制终端、随系统启动而启动、随系统关闭而退出的特殊进程，负责提供各种系统服务和功能
+ 系统调用：为用户程序提供一组受控的、安全的接口，允许用户程序操作系统内核运行更高权限的资源，是用户态主动进入内核态的唯一标准入口
+ 虚拟化：将单个计算机硬件抽象成多个不同的执行环境，造成有多个硬件的错觉
+ 单步：将 TF 位设置位 1，CPU 每执行一个指令就触发一次陷阱，可用于逐行调试代码
+ 链接器：将编译器生成的多个目标文件以及库文件组合成一个完整的可执行文件
+ 加载器：将可执行文件从磁盘加载到内存中，并为其分配必要的资源，最终让程序能够被 CPU 执行，把程序从文件变成进程
+ 机制：系统为实现某种功能而提供的底层操作、数据结构或硬件支持（如何做）
+ 策略：系统在特定目标下，决定如何使用机制的规则、算法或决策逻辑（做什么）
+ 耦合：用来衡量系统中不同模块之间相互依赖的紧密程度，耦合度越高，一个模块的变化越容易影响其他模块
+ HAL：位于操作系统内核与物理硬件之间的软件层，目的是向上层提供统一的硬件访问接口，屏蔽底层硬件的差异

# 进程
+ 程序：一组预先编写好的、有序的指令集合。它存储在磁盘、固态硬盘或其他非易失性存储介质中，是一个静态的、被动的实体
+ 进程：程序的一次执行过程，是系统进行资源分配和调度的基本单位 
+ 线程：一个进程至少包含一个线程（主线程），也可以包含多个线程，线程是CPU调度的基本单位，这意味着CPU真正执行的是线程不是进程
+ 协程：协程的创建、切换、销毁完全在用户空间完成，开销低
+ PCB：操作系统中用于描述和控制进程运行的一个专门数据结构，它是进程存在的唯一标志 
+ 交换：将处于阻塞或等待状态的进程从内存移动到磁盘的交换空间，将磁盘交换区中处于就绪状态的进程重新加载回内存，使其恢复执行
+ 上下文切换：操作系统内核将 CPU 从一个进程切换到另一个进程，保存当前进程的执行状态，恢复目标进程的先前状态
+ 僵尸进程：已经终止运行，但仍然保留 PCB，等待父进程回收
+ 孤儿进程：父进程已经结束，子进程还在进行，会被 init 进程 PID = 1 自动收养并回收资源
+ 进程间通信：管道、消息队列、共享内存、信号量、信号、套接字
+ 写时复制：当多个任务需要访问同一份资源时先共享同一份副本，只有当某个任务试图修改这份资源时，才真正为其创建一个独立的副本
+ 异步线程：父线程创建子线程后，父线程与子线程并发执行相互独立
+ 同步线程：父线程创建子线程后，父线程要等待子线程终止后才继续执行
+ 隐式线程：不由程序员显式创建和管理，而是由编译器、运行时环境或并行编程模型自动创建和管理的线程
+ 线程池：一种预先创建并管理一组线程的技术，用于在需要时重复使用这些线程来执行多个任务，避免频繁创建和销毁线程的开销
+ 调度算法：先到先服务、最短作业优先、时间片轮转、优先级调度、多级队列、多级反馈队列
+ 实时调度算法：单调速率、最早截止期限优先、比例分享、POSIX 实时调度
+ 饥饿：进程或线程因长时间未获得所需资源而无法运行
+ 调度程序：负责决定下一个哪个进程获得CPU使用权的模块
+ 分派程序：一个模块，将 CPU 的控制权交给调度程序选择的进程
+ 分派延迟：停止一个进程到启动另一个进程之间的时间开销

# 同步
+ 临界区：程序中访问共享资源的那一段代码，这些共享资源在同一时刻只能被一个线程或进程访问
