# 计算机网络
+ OSI 网络模型
  物理层、数据链路层、网络层、传输层、会话层、表示层、应用层
+ TCP/IP 网络模型
  网络接口层、网络层、传输层、应用层
+ 物理层的作用
  将原始数据（比特流）转换为可以在物理介质上传输的信号，并在接收端将信号还原为比特流
+ 不同传输介质的区别
  |介质|场景|特点|
  |:-:|:-:|:-:|
  |双绞线|家庭路由器、数据中心服务器短距离连接|便宜、易用、距离短|
  |同轴电缆|有线电视网络、宽带接入|抗干扰好|
  |光纤|长距离传输、海底光缆、跨城市互联网|极快、距离远、不受电磁干扰|
  |无线|手机、笔记本、物联网设备|灵活移动、但易受干扰|
+ TCP 与 UDP 的区别
  TCP 是面向连接、可靠、有序；UDP 是无连接、不可靠、无序
+ TCP 三次握手
  客户端向服务器发送一个SYN报文，服务器收到客户端的SYN报文后服务器回复一个SYN+ACK报文，客户端收到服务器的SYN+ACK报文后发送一个ACK确认报文
+ TCP 四次挥手
  客户端完成数据发送后主动向服务器发送一个FIN报文，服务器收到客户端的FIN报文后立即回复一个ACK确认报文，
  服务器完成所有数据发送后主动向客户端发送一个FIN报文，客户端收到服务器的FIN报文后回复一个ACK确认报文
