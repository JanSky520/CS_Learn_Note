# 概论
## 导论
### 学习笔记
+ 总线：总线是连接计算机系统中多个部件（如 CPU、内存、I/O 设备）之间进行数据传输的一组公共通信线路
+ 中断：处理器的一种异步响应机制，由外部硬件或内部异常条件触发，暂停当前执行序列，执行预设中断处理程序，完成后恢复
+ 陷阱：处理器的一种同步机制，由正在执行的程序主动触发，与中断一样从用户态切换到内核态，处理完后返回用户态
+ 直接内存访问（DMA）：允许硬件设备绕过 CPU，直接与系统内存进行数据传输
+ 系统守护进程：在后台运行、没有控制终端、随系统启动而启动、随系统关闭而退出的特殊进程，负责提供各种系统服务和功能
+ 系统调用：为用户程序提供一组受控的、安全的接口，允许用户程序操作系统内核运行更高权限的资源，是用户态主动进入内核态的唯一标准入口
+ 虚拟化：将单个计算机硬件抽象成多个不同的执行环境，造成有多个硬件的错觉
### 练习
1. 提供用户与计算机硬件之间的接口、管理计算机资源、控制程序的执行
2. 出现故障、死机、程序报错等时候需要“浪费”资源，操作系统的设计是权衡的艺术，资源浪费是否合理取决于更高层次的系统目标
3. 资源受限环境的高效管理、并发与同步的可靠性、中断处理的实时性、硬件兼容性与低层优化
4. 不应该包含，增加操作系统的复杂度，不易于维护，操作系统只提供基础功能，剩下的交给软件开发者即可
5. 模式位（0 表示内核，1 表示用户）；特权指令限制、内存隔离、系统调用门
6. a、c、e、f、g、h
7. 灵活性受限、功能限制
8. 安全隔离、虚拟化支持
9. 当前时间 = 启动时间 + (中断次数 * 中断间隔)
10. 解决速度不匹配、减少重复计算；性能瓶颈、低效性；复杂度增加、成本高昂
11. CS 模型是服务器提供服务，客户端发起请求；P2P 模型是所有节点既是客户端又是服务器
### 习题
12. 集群是多台独立计算机通过网络连接、多处理器是共享内存的多个 CPU；机器间定期通信确认存活状态
13. 共享磁盘架构：优点是数据一致性容易维护，实现简单；缺点是磁盘成为单点故障，扩展性受限  
    数据复制架构：优点是无单点故障，读取性能高；缺点是写操作需同步，网络开销大，一致性维护复杂
14. 中断是用来处理外部事件的；中断主要由外部硬件引起，陷阱主要由程序触发；用户可以随意触发陷阱，因为这是用户程序主动请求的
15. uptime_seconds = jiffies / HZ
16. a. 当 I/O 设备传输数据时，CPU 对 DMA 编程，设置源/目标地址等，随后 DMA 控制器接管总线控制权  
    b. CPU 通过中断机制得知 DMA 操作结束  
    c. 会产生内存访问冲突
17. 不可以构建一个真正安全的操作系统,操作系统的安全性依赖硬件支持的隔离保护机制
18. 为了平衡访问速度和数据一致性
19. f->c->a->e->d->g->b
20. 在 SMP 系统中，内存数据与本地缓存数据不一致主要源于多核并发操作
21. a. 写穿透策略、写回策略  
    b. 写无效协议、写更新协议  
    c. 2PC、最终一致性
22. 基址寄存器存放进程的内存空间的物理起始地址，界限寄存器存放空间的大小或偏移量，当执行一条指令时硬件会检查物理地址是否在范围内，超出会触发异常
23. a. LAN  
    b. WAN  
    c. LAN 或 WAN
24. 资源严格受限、多样化的传感器与 I/O、电源管理与热管理、安全与隐私模型
25. 可扩展性强、成本低廉、鲁棒性与容错性高、资源共享直接高效
26. 文件共享、即使通讯与 VoIP、加密货币与区块链、分布式计算
27. 源代码公开透明，安全风险小；资金容易短缺，用爱发电
